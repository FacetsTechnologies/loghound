addMsgFilter(newFilter)
Adds a filter which extends LogHoundMessageFilter to the message filter array. You can add your own filters to the standard set by creating your own LogHoundMessageFilter implementation and submitting it to this function.

Each filter MUST have a unique ID. Submitting a filter with an existing ID will replace the existing filter with the submitted filter.

Parameters:
{LogHoundMessageFilter} newFilter
The message filter to add to the internal array.

setKillSwitch(killSwitch)
When set to true Log Hound will be completely blocked from being initialised via the doSetup() function. This function is usually used to prevent Log Hound from initilising on a global basis in instances where your code is in an environment that you do not want the Log Hound UI to be seen: the public version of your site as opposed to the development version of your site, for instance.
Parameters:
{boolean} killSwitch
Set to true to block Log Hound from being initialised.

setMessageLayout(layout)
Sets the message layout display to either 'brief' or 'detail'. If Log Hound is active, the message layout in the UI will then be changed to the indicated layout. If no argument or null is passed to this function, the message layout will be toggled between the 'brief' and 'detail' layouts.

The layout can be set before Log Hound is set up.

Parameters:
{String|null} layout
Can be the values 'brief', 'display', or you can pass no value at all.
Returns:
true if the function call resulted in the message layout being changed, otherwise false.
{boolean} setMessagePaneSize(size)
Sets the message pane size to a specific height. The upper and lower bounds for the height are 600 and 75 respectively. Calls argumenting sizes outside those bounds will be ignored.
Parameters:
{number} size
The new height to set the message pane to.
Returns:
{boolean} true if the message pane was adjusted, otherwise false.
setShadeMode(shade)
Controls whether or not the Log Hound user interface is in "shade mode", where only the header strip is visible in order to stay out of the user's way. If no argument or null is passed, this method acts as a toggle.
Parameters:
{boolean|null} shade
true if the user interface should roll up into shade mode, false for the user interface to roll out. null to toggle the mode.
show(show)
If set to a value that equates to 'true', the Log Hound UI will be shown on the page. If set to a 'false' value, Log Hound will be hidden. If no value is argumented, this function acts as a toggle. Please note that while hidden, Log Hound is still active and will continue to log messages.
Parameters:
{boolean|String} show
A value that equates to 'true'

Values that equate to true:

  • true
  • 'true'
  • 'show'
All other values except null equate to false.
showMessageLevel(level, show)
Shows or hides all messages for the specified level. If the 'show' argument is null or not passed in, this function will toggle the visibility of the specified level messages.
Parameters:
{String|Number|LogHoundLevel} level
The log level to show, hide, or toggle.
{String|boolean} show
true if you want the messages of the argumented level to be visible, false if you want to hide the level messages, or null or do not pass a value to toggle the target level's messages.

Values that equate to true:

  • true
  • 'true'
  • 'show'
All other values except null equate to false.
toggleCtrlPanel(cmd)
Parameters:
{String} cmd
Can be one of four strings: display, hide, toggle, or restore.
  • display: Displays the panel.
  • hide: Hides the panel.
  • toggle: Toggles the panel from the saved state.
  • restore: Restores the panel to the saved state.
